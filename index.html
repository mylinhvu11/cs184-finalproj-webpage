Project Summary

For our project, we plan on implementing an animation that utilizes spacial keyframes to add movement to a scene. This also speeds up the process of animators adding movements to their characters since they can use less keyframes and add fluidity to the animation. Our project group is Riya Kumar, Mylinh Vu, and Ali.

Problem Description

Here you should provide the context for your idea. Describe the problem that you are trying to solve, why it is important, where it is challenging. Give us a general idea on how you are going to solve it.

Often times, animators have to implement many frame images to get their idea and art to come to life with movement. With their drawings, there are often times that the movemnet doesn't seem fluid which takes away from the scene so this implementation should ease these problems and allow it to be more efficient to build an animation. We are essentially trying to interpolate the points between images to create a fluid motion for the animation. This is important to allow more people to connect with the art of animation and ease the creativity process with the increase effeciency of making an animation. The challenging part will be deciding how to interpolate these points and how to constanly update the points to update on the screen with fluidity. Between two images could have a lot of subimages to create the animation and we believe the tough part will be interpolating the points at the correct rates to achieve the fluid animation. To implement our idea, we plan on utilizing radial basis interpolation that will help give the interpolation points that should achieve the fluidity and orthonormalization to ensure the rotation matrix for certain movmenets is correct so the images don't warp and appear correctly. If we have time, we would also like to implement reverse kinematics to add the lifelike and realistic physical animations that would normally occur in real life.

Goals and Deliverables

What We Plan to Deliver

describe what you believe you must accomplish to have a successful project and achieve the grade you expect (i.e. your baseline plan -- planning for some unexpected problems would make sense)

We are going to create a performance driven animation using spacial key framing. The character will be a penguin. We are going to have 4 key frames so the penguin will have four position to rotate between to simulate motion. Then penguin will follow the movement of the mouse around the screen with fluid motion. The program will follow a user's mouse inputs in real-time so the resulting animation will be an expression of the user's movements.

What We Hope to Deliver

We aim to provide 4 key frames and hope to get a 2-3 second loop that is rendered at a smooth 30FPS with avaiable tunability in regards to frame rate. We want to create a penguin model that is highly detailed with a large number of joints so our reverse kinematics model has enough complexity to make a variety of different actions without running into issues such as self intersection or jittering. The primary goal is to build a system that manipulates the model between given key frames that does not violate the structural skeleton of the model. We would also like to implement ideas such as textures and lighting to make the models presence in the space grounded in reality. We also hope to build some interactive functionality where we can shift between different sets of key frames to make the model move in a predictable and smooth manner. Having multiple models exist in the same space following a mouse is an idea we would like to explore further and if we can get them to interact with each other using key frames that we create.
